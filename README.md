# 📖 Word Extractor — 英文から語彙を整理して学習に活かすための Web ツール

![demo](screenshots/demo_optimized.gif)

英語の長文を読む際に、「知らない単語だけを整理して学習したい」と思った経験から開発した  
**PDF / Web ページから英単語を抽出し、一覧表示できる語彙学習支援アプリ**です。

全文翻訳ではなく、**語彙の整理に特化**している点が特徴です。

---

## 🎯 背景と目的（Why）
英語の学習中、文章内に出てくる **“新出単語をひとつずつ調べる作業”** に時間がかかっていました。

既存の翻訳ツールは
- 文章全体の翻訳 → 得意
- **語彙のみを抽出し整理する → 不得意**

という課題があったため、

> 「長文 → 単語 → 意味 → 学習」まで、  
> **一連の流れをシンプルにできるツールを作ろう**

という考えで本アプリを作成しました。

---

## 🛸 主な機能
| 機能 | 詳細 |
|---|---|
| PDF から英単語を抽出 | PDFBox を使用してテキスト抽出後、正規表現で不要な文字を除去 |
| Webページから抽出 | Jsoup で HTML を解析し、タグやリンクを除外して精度を向上 |
| A〜Z のカテゴリ表示 | 語彙を視覚的に整理しやすい UI |
| 重複排除 | `Set` により効率的にユニーク化を実装 |
| 発音 / 英語定義の参照 | 単語の意味を画面内で確認可能 |
| DeepL 和訳リンク | 英語定義 → 和訳に1クリックで移動できる動線 |

---

## 💡 UI / UX の工夫
- 学習中に迷わないよう **画面遷移を最小限** に設計
- 単語の意味は **モーダル表示** で、位置情報を失わない
- Bootstrap による **視認性と操作性のバランス**

> 「使う人が迷わないツール」になることを重視しました。

---

## 🛠 技術構成
| 項目 | 使用技術 | 選定理由 |
|---|---|---|
| Backend | Spring Boot (Java) | MVCで構造を整理しやすい |
| Template Engine | Thymeleaf | HTMLを自然に扱え、メンテ容易 |
| Frontend | Bootstrap 5 | UI実装が早く、レイアウト再現性が高い |
| Parsing | Jsoup / PDFBox | Web / PDF の抽出に適している |
| Build | Maven | 依存管理と再現性が明確 |

---

## 🔧 開発で工夫した点（再現性・現場対応力）
- **抽出精度の向上**  
  PDF / HTML には余計な文字が含まれるため、  
  正規表現・ノイズ除去のルールを改善し、再現性を確保

- **処理のシンプル化と保守性**  
  複雑なロジックは避け、  
  「他の人が読んで理解できる構造」を優先

- **トラブル対応を想定したログ分離**  
  予期せぬ入力に対しても例外処理と通知を整理

> **「動くものを安定して運用できること」を大切にする姿勢**が反映されています。
